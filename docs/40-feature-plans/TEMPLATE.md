# Feature Plan Template

把这份模板复制成新的规划文档，只保留和当前功能有关的部分。

## 1. 背景

- 当前问题是什么
- 为什么不能直接在现有实现上小修
- 这份规划覆盖哪些 feature / 系统

## 2. 目标与非目标

### 2.1 目标

- 明确这次要做到什么

### 2.2 非目标

- 明确这次不做什么

## 3. 当前实现

- 当前 source of truth 在哪里
- 当前链路如何工作
- 当前已知约束和坑点

## 4. 方案设计

### 4.1 数据模型

- 表结构 / Freezed 模型 / JSON 协议

### 4.2 状态与流程

- Provider / Notifier / Repository 如何协作

### 4.3 UI 与交互

- 页面
- 弹窗
- 错误态
- 平台差异

## 5. 边界与失败模式

- 登录态缺失
- 网络失败
- 旧数据兼容
- 并发 / 重试
- 平台差异

## 6. 实施步骤

1. 第一阶段做什么
2. 第二阶段做什么
3. 最后如何收口

## 7. 验证计划

- 最低自动化验证
- 手测场景
- 回归风险点

## 8. 文档落地计划

功能完成后，要把稳定结论提炼到哪些主线文档：

- `docs/10-project/...`
- `docs/20-workflows/...`
- `docs/30-reference/...`

原始规划文档完成使命后，移入 `docs/90-archive/feature-plans/`。
